# spec-055 Вылазки — доработки (по тестам Andrey, 2026-07-08)

Батч доработок поверх отгруженной 055. Ветка `claude/spec-055-doraboтки` от
`origin/main`. Собирается волнами агентов (W1 данные → W2 UI).

## 5 пунктов (Andrey)

1. **Дата** — пикер: день 1–30 месяца странствий + старт HH:MM.
2. **Длительность** — HH:MM (сколько заняла).
3. **Набор PC** — участники (уже есть в ход-шите) + семя «времени»: прогон хранит
   день + старт + длительность; время PC = сумма длительностей его вылазок (семя spec-057).
4. **Награда-ресурсы** — предмет каталога с номиналом + продажа из общака.
5. **UX** — вынести ⋮ (Балансы/Вылазки/Каталог) в кнопки + проход по меню.
   (Полный редизайн — потом в Claude Design; сейчас «нагромождаем функционал».)

## Решения (AskUserQuestion, 2026-07-08)

- **Ресурс** = постоянный предмет каталога, категория «Ресурс» (`node_type='item'`,
  `item_attributes.category_slug='resource'`, `price_gp=номинал`).
- **Продажа** — и в /tg (общак-экран, игроки), и на десктопе (accounting/stash, ДМ);
  **с выбором количества**.
- **Календарь** — СТРОГОЕ окно: петля = месяц странствий, 30 дней, **день 1 02:00 →
  день 31 02:00** (= 02:00 месяца рдения). Валидировать старт+длительность в окне.

## Модель данных

- `expedition_runs` + `start_minute int` (0..1439, минуты внутри дня), `duration_minute int`. (миг 125)
- Категория `scope='item', slug='resource', label='Ресурс'` (сид, миг 125).
- `runExpedition` + `startMinute?`, `durationMinute?` (валидация окна).
- `createResourceItem({campaignId, name, priceGp}) → {itemNodeId, name}` (дедуп по имени+категории).
- `sellStashResource({campaignId, itemNodeId, qty, loopNumber, dayInLoop}) → {soldGp}`
  (выемка предмета из общака −qty + доход +price×qty, авто-апрув, событие `income`).
- Календарь: `lib/expedition-calendar.ts` — `validateExpeditionWindow`, `LOOP_DAYS=30`,
  `LOOP_START_HOUR=2`, `LOOP_START_ABS_MIN=120`, `LOOP_END_ABS_MIN=43320`.

## Формат события ленты `'expedition'` (залочен Andrey)

Событие несёт: loopNumber, dayInLoop, startMinute, durationMinute, участники, цель,
расходники (list), награду (предметы с опц. `priceGp` для ресурсов, деньги).

```
🧭 <b>Вылазка</b>
Петля {N}, День {X} · с {HH:MM} по {HH:MM}
{участники через «, », «и» перед последним} отправились на вылазку «{Цель}»[, потратили:]
[{расходники: Name ×qty, ...}]           ← только если были расходники
<b>В общак добавлены:</b>                 ← только если есть награда
{ресурс: Name ×qty (номинал×qty зм), обычный лут: Name ×qty, деньги: N зм}
```

Правила:
- Время «с HH:MM по HH:MM» = старт → старт+длительность. Многодневная → «День X → День Y».
- «отправились» — универсально (пол PC неизвестен, как «собрал(а)»).
- Ресурсы (у предмета есть price_gp) — цена **за строку** (номинал×qty) в скобках; обычный лут — без цены.
- Расходники — только список, без суммы (решение Andrey «оставить без суммы»).
- Шапка «🧭 <b>Вылазка</b>» — оставить.
