# spec-055 Вылазки — доработки (по тестам Andrey, 2026-07-08)

Батч доработок поверх отгруженной 055. Ветка `claude/spec-055-doraboтки` от
`origin/main`. Собирается волнами агентов (W1 данные → W2 UI).

## 5 пунктов (Andrey)

1. **Дата** — пикер: день 1–30 месяца странствий + старт HH:MM.
2. **Длительность** — HH:MM (сколько заняла).
3. **Набор PC** — участники (уже есть в ход-шите) + семя «времени»: прогон хранит
   день + старт + длительность; время PC = сумма длительностей его вылазок (семя spec-057).
4. **Награда-ресурсы** — предмет каталога с номиналом + продажа из общака.
5. **UX** — вынести ⋮ (Балансы/Вылазки/Каталог) в кнопки + проход по меню.
   (Полный редизайн — потом в Claude Design; сейчас «нагромождаем функционал».)

## Решения (AskUserQuestion, 2026-07-08)

- **Ресурс** = постоянный предмет каталога, категория «Ресурс» (`node_type='item'`,
  `item_attributes.category_slug='resource'`, `price_gp=номинал`).
- **Продажа** — и в /tg (общак-экран, игроки), и на десктопе (accounting/stash, ДМ);
  **с выбором количества**.
- **Календарь** — СТРОГОЕ окно: петля = месяц странствий, 30 дней, **день 1 02:00 →
  день 31 02:00** (= 02:00 месяца рдения). Валидировать старт+длительность в окне.

## Модель данных

- `expedition_runs` + `start_minute int` (0..1439, минуты внутри дня), `duration_minute int`. (миг 125)
- Категория `scope='item', slug='resource', label='Ресурс'` (сид, миг 125).
- `runExpedition` + `startMinute?`, `durationMinute?` (валидация окна).
- `createResourceItem({campaignId, name, priceGp}) → {itemNodeId, name}` (дедуп по имени+категории).
- `sellStashResource({campaignId, itemNodeId, qty, loopNumber, dayInLoop}) → {soldGp}`
  (выемка предмета из общака −qty + доход +price×qty, авто-апрув, событие `income`).
- Календарь: `lib/expedition-calendar.ts` — `validateExpeditionWindow`, `LOOP_DAYS=30`,
  `LOOP_START_HOUR=2`, `LOOP_START_ABS_MIN=120`, `LOOP_END_ABS_MIN=43320`.

## Формат события ленты `'expedition'` (залочен Andrey)

Событие несёт: loopNumber, dayInLoop, startMinute, durationMinute, участники, цель,
расходники (list), награду (предметы с опц. `priceGp` для ресурсов, деньги).

```
🧭 <b>Вылазка</b>
Петля {N}, День {X} · с {HH:MM} по {HH:MM}
{участники через «, », «и» перед последним} отправились на вылазку «{Цель}»[, потратили:]
[{расходники: Name ×qty, ...}]           ← только если были расходники
<b>В общак добавлены:</b>                 ← только если есть награда
{ресурс: Name ×qty (номинал×qty зм), обычный лут: Name ×qty, деньги: N зм}
```

Правила:
- Время «с HH:MM по HH:MM» = старт → старт+длительность. Многодневная → «День X → День Y».
- «отправились» — универсально (пол PC неизвестен, как «собрал(а)»).
- Ресурсы (у предмета есть price_gp) — цена **за строку** (номинал×qty) в скобках; обычный лут — без цены.
- Расходники — только список, без суммы (решение Andrey «оставить без суммы»).
- Шапка «🧭 <b>Вылазка</b>» — оставить.

# Round-2 (по тестам Andrey, 2026-07-08)

Ветка `claude/spec-055-dorabotki-r2` от `origin/main`. По тестам round-1 «вообще
неправильно сделано»: шаблон вылазки держал только имя/описание/расходники, а лут,
участники и время вбивались в КАЖДЫЙ прогон.

## Модель (залочена Andrey)

**Шаблон вылазки = ПОЛНЫЙ набор дефолтов; форма прогона подставляет всё, но ВСЁ
редактируемо** («дефолты, не локи», гибкая система). Обычно «Сходить» — один-тап,
при нужде правишь любое поле (включая лут).

| Что | Где живёт |
|---|---|
| Награда (деньги / предметы / ресурсы с номиналом) | **шаблон** (стабильна между петлями) |
| Ростер (участники) | шаблон = дефолт → прогон правит |
| Старт + длительность | шаблон = дефолт → прогон правит |
| Расходники | шаблон = «потенциальные» → прогон правит |
| **День петли** | только per-run (в шаблоне НЕ хранится) |

## 6 пунктов

1. **Шаблон=дефолты** — миграция **126**: `expeditions` += `reward_money_gp`,
   `reward_items`, `default_participant_node_ids`, `default_start_minute`,
   `default_duration_minute`. `default_duration_ticks` (124) → DEPRECATED (единица
   прогонов — минуты после 125). `runExpedition` НЕ меняется — форма подставляет.
2. **Пикер участников** — компактный тёмный (паттерн `components/participants-picker.tsx`,
   перекрашен под /tg): свёрнутый триггер → поиск + чекбоксы, выбранные наверх.
   Заменил высокий грид всех PC (по тестам «пришлось вытягивать вверх»).
3. **Мобила — ничего не скрывается** — `Sheet` `max-h-[90vh] overflow-y-auto`,
   длинная форма скроллится, сабмит достижим (/tg = мобильный продукт игроков).
4. **Sheet без закрытия по бэкдропу + «← Назад»** — мис-тап больше не сносит форму;
   глобально на ВСЕ /tg-шиты (Записать/Перевод/Купить/наборы/вылазки), у каждого
   остаётся явный выход.
5. **Общак в мастер-сообщении** — ресурсы с номиналом в скобках
   (`Листва Ивы ×1 (3000 зм)`) + рядом с `Общак: X зм` через запятую сумма
   непроданных ресурсов (`, +Y зм в ресурсах`). `ledger-master-format.ts` +
   `composeMasterState` (резолв номинала по имени через `getStashResourceHoldingsTg`).
6. **#5 Десктоп `/accounting`** (выбор Andrey «причесать существующее», не новая
   лента): группировка строк одной вылазки (общий `transfer_group_id`) в одну
   читаемую запись + фильтр по типу. Прецедент — `dedupTransferPairs`.

## Решения / расхождения

- **Лут НЕ read-only** (Andrey: «всё можно изменить»): форма прогона подставляет
  награду из шаблона, но она правится как и всё остальное.
- **Ad-hoc прогон** (без шаблона) — прежний путь с инлайн-вводом награды.
- **День петли** — дефолт 1 (текущего дня нет в пропсах `ExpeditionsScreen`).
- **Ресурс шаблона** при ре-редактировании виден в «Предметах», не «Ресурсах»
  (на сохранении find-or-create'ится в каталожный item) — косметика, round-3.
