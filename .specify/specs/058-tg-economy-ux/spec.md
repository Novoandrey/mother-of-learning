# Feature Specification: UX-переработка /tg-экономики (spec-058)

**Feature Branch**: `claude/spec-058-tg-ux`
**Created**: 2026-07-09
**Status**: 📗 Specify — бриф Andrey + аудит текущего /tg + ресёрч паттернов.
Дизайн-развилки — на решении Andrey (см. §Развилки).

## Бриф (Andrey, 2026-07-09)

Игрокам должно быть ЛЕГКО делать транзакции. Канонический путь: **игрок пишет
в чат, что произошло** («потратил 5 зм», «купил зелье», «надел кольчугу») —
и должен так же легко провести это в приложении. Требования: (1) ~все
транзакции из ОДНОЙ зоны; (2) потом — надеть вещи в инвентаре; (3) собрать
раскиданный функционал, унифицировать; (4) вдохновляться RPG-играми и
геймдизайн-книгами. Ревью — отдельным агентом (`.claude/agents/ux-auditor.md`).

## Дизайн-тезис

**Интерфейс говорит языком намерений игрока (глаголы), а не бухгалтерской
таксономии (сущности).** Сейчас разделы = сущности (Деньги/Сумка/Общак/
Вылазки/Крафт) и 5 разных UI-паттернов; станет: зона действий с глаголами
(«Потратил», «Купил», «Передал», «Надел»), сходящимися в единый пайплайн
«короткая форма → превью → подтверждение».

Обоснование — консенсус игрового и финтех-UX (research.md, 9 паттернов,
30+ источников): Persona 5 verb-based команды; финтех-хабы (Revolut: баланс +
плитка глаголов); tap-to-equip вместо drag-n-drop (уроки мобильного Slay the
Spire); Schell/Norman: частое действие — 1–2 шага, без размышлений.
Осознанный отказ: радиальные меню BG3 (паттерн контроллера, не пальца).

## Диагноз текущего UI (audit.md — полная карта)

- **5 паттернов** для экономических действий: FAB+шит / кнопка+шит / отдельный
  экран+шит / инлайн-строка / экран-хаб.
- **Одно намерение — разные дома**: «взять из общака» деньги = экран Общак,
  предметы = Сумка→Переместить; «положил в общак» — два разных пути.
- **Покупка живёт под «Сумкой»** — игрок с намерением «трачу деньги» её не
  находит. Кредит — на дне экрана Снаряжения.
- До **7 кнопок** лончера + дубль-корни (CharacterList и PcHome).
- Мёртвое: RequestsScreen недостижим; идентичный тернарник dirOptions;
  `dayInLoop: 1` зашит в ~11 местах.
- Нет навигационного стека — `onBack` вручную и неконсистентно.
- ✅ **Слой server actions + lib/queries/*-tg переиспользуется без правок.**
  Примитивы (Sheet R2, SegToggle, IntInput, WalletCard, FeedList, пикеры,
  Funding/Reward-редакторы) — база нового UI.

## Целевая структура (предложение → развилка №1)

Нижний таб-бар, 3 таба:

1. **⚡ Действие** (корень) — «одна зона»: чипы последних операций игрока
   (тап = предзаполненная форма) + плитка глаголов ≤6: **Потратил · Получил ·
   Купил · Передал · Продал · Ещё** («Ещё»: кредит, стартовый набор, заявки).
   Каждый глагол → короткий Sheet (паттерн R2) → превью → сабмит.
   Поле «Что произошло?» (NL-ввод) — развилка №2.
2. **🎒 Персонаж** — кошелёк + «Надето» (компактные слоты, tap-to-equip) +
   «Сумка» + лента своих движений. Тап по предмету → шит глаголов: Надеть/
   Снять · Передать · В общак · Продать — те же формы, тот же пайплайн.
3. **🏰 Партия** — общак (кошелёк+предметы+положить/забрать+ресурсы),
   Вылазки, Крафт, Балансы всех. Каталог/вики — кнопкой в шапке.

Пайплайн действия един: глагол → форма (минимум полей, умные дефолты) →
превью строки («−50 зм · зелье лечения ×1 · за свои») → MainButton-стиль
сабмит → тост + обновление. Направления переводов разложены по глаголам
(«Передал» = кому/куда; «Взял» из общака — глагол в Партии и в «Ещё»),
мега-TransferSheet с 6 режимами уходит.

## Унификация по пути

- Единый навигационный стек (View[] вместо ручных onBack).
- Выкинуть: RequestsScreen (мёртв), тернарник dirOptions, дубль-корни.
- `dayInLoop` — оставить конвенцию «1» для быстрых действий v1 (осознанно,
  день — материя спеки 057), НО в одном месте (константа с комментом).
- Продажа предмета из сумки PC — новое (сейчас только ресурсы общака) —
  развилка №4.

## Развилки — РЕШЕНЫ (Andrey, 2026-07-09)

1. **Каркас: 3 таба** (⚡ Действие / 🎒 Персонаж / 🏰 Партия), нижний таб-бар.
2. **NL-ввод: v1 без него** — чипы последних операций + глаголы; парсер —
   отдельный апгрейд после обкатки каркаса.
3. **Глубина: ПОЛНАЯ ПЕРЕБОРКА** — все экономические формы заново под единый
   пайплайн «глагол → форма → превью → сабмит», `ledger-app.tsx` (4941 строка)
   распиливается на модули. Уточнение: формы вылазок (055 R2) и крафта (056)
   УЖЕ в целевом паттерне — переносятся в таб Партия без переписывания;
   переборка = формы 044/052 (записать/перевод/покупка/общак/наборы/
   снаряжение).
4. **Продажа из сумки PC: включить** — новый server action `sellPcItem`
   (item-out −qty + доход, канон денежных строк; цена — из каталога с
   правкой), глагол «Продал» в зоне действий.

## Success Criteria

- SC-001: каждая типовая фраза игрока (потратил/получил/купил/передал/надел/
  взял из общака/продал) — ≤3 тапа от корня до сабмита, все из одной зоны.
- SC-002: один UI-паттерн для всех экономических форм (Sheet R2 + превью).
- SC-003: «надел кольчугу» — 2 тапа из таба Персонаж; слоты видны сразу.
- SC-004: ни одного мёртвого экрана/поля; навигация — стек, «назад»
  консистентен.
- SC-005: ux-auditor даёт вердикт «ship» по чек-листу фраз игрока.

## Out of scope (v1)

Десктоп-поверхности; NL-парсер LLM; пересмотр самих server actions;
день петли в быстрых действиях (спека 057); редизайн вики/каталога.
