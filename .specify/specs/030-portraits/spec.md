## Context

`character_portraits` (mig 116, spec-046) уже one-to-many и рассчитана на будущую карусель: «next spec adds upload + carousel + per-portrait metadata (loop / inspiration / description), no migration» — эта спека и есть тот «next spec». В проде на нодах-персонажах рендерится один primary-портрет только в /tg (карточка PC); на десктопе портретов нет вовсе. Арты (нейросетевые) выгружены Андреем — `AI-Art.zip` (261 МБ, 138 png; безымянные `ChatGPT-*` игнорируются, остаётся 126 именованных). Имя файла = title ноды.

Ключевой факт (сверено с живой БД): персонажи разнесены по типам `character` (32 = PC), `npc` (193), `creature` (64). Прежний `seed-portraits.ts` матчил только `character` — то есть почти все арты (NPC) пропускал. Это главная доработка импортёра.

## Match-анализ 126 именованных файлов (по БД, 2026-07-03)

- **110** — точный матч по title (character/npc/creature).
- **9** — через алиасы (`Имайа→Имайя Курошка`, `Мерега→Мерега, дочь Агонии`, `Новизна→Исполненная… (Новизна)`, `Оран Скарна _Ворчун_→"Ворчун"`, и т.д. — все разрешены против БД).
- **4** — карусель: `Кватач-Ичл, генерал/лич/маскировка/человек` → одна нода «Кватач-Ичл» (4 портрета, подпись = часть после запятой).
- **2** — требуют решения DM (см. Clarifications C-01, C-02).
- Итого ~123 арта размещаются, 2 отложены.

## User Scenarios & Testing

### User Story 1 — Видеть арты персонажа/NPC на десктопе (P1) ✅ Phase 1

DM открывает ноду персонажа/NPC в каталоге и видит портрет; если артов несколько — листает карусель (‹ n/N ›, подписи, точки). Ноды без артов выглядят как раньше.

**Acceptance**: (1) нода с 1 артом → портрет под заголовком; (2) «Кватач-Ичл» с 4 → карусель с подписями; (3) нода без артов → карусель не рендерится, страница как прежде.

### User Story 2 — Каталог неписей + статья в Telegram (P1) — Phase 2

Игрок в /tg открывает каталог, ищет неписа, открывает — видит карусель артов + markdown-статью (лор). Только чтение; visibility нет (все члены кампании видят всё).

**Acceptance**: (1) список персонажей/NPC/существ с мини-аватарами + поиск по имени; (2) открытая нода → тёмная карусель + отрендеренный markdown; (3) пустая статья → мягкий плейсхолдер.

### User Story 3 — Управление портретами из приложения (P2) — Phase 3

DM (и владелец PC для своего) добавляет/заменяет/удаляет портрет, задаёт главный и подпись прямо в приложении, без запуска скрипта. Загрузка в R2 из браузера.

**Acceptance**: (1) добавить портрет к ноде → появляется в карусели; (2) сменить главный / порядок / подпись; (3) удалить.

## Functional Requirements

- **FR-001** Импортёр матчит по title среди типов `character`+`npc`+`creature`. ✅
- **FR-002** Алиасы + underscore→кавычки; неоднозначные (title под >1 нодой) — через TYPE_PIN, иначе пропуск с пометкой; безымянные `ChatGPT-*` — пропуск. ✅
- **FR-003** «База, подпись» без точного матча, где База — точный title → карусель этой ноды (подпись = хвост); настоящие запятые в title матчатся точным путём раньше. ✅
- **FR-004** `sort_order` + `caption` per-портрет; ровно один `is_primary` на ноду (partial-unique индекс, mig 121). ✅
- **FR-005** Импортёр идемпотентен, dry-run по умолчанию, `--commit` для записи; ключ R2 одиночного = `<node_id><ext>`, карусельного = `<node_id>-<i><ext>`. ✅
- **FR-006** Десктоп: карусель на странице ноды для character/npc/creature; пусто → ничего. ✅
- **FR-007** `portraitUrl`/порядок/тип — общий модуль `lib/portraits.ts` (десктоп + /tg). ✅
- **FR-010** /tg: каталог (список + поиск) персонажей/NPC/существ; открытие → карусель + markdown-статья (read). — Phase 2
- **FR-011** /tg-карусель тёмная, тач-дружелюбная; markdown через react-markdown+remark-gfm. — Phase 2
- **FR-020** Загрузка/замена/удаление/set-primary/подпись/порядок из приложения (desktop + /tg), R2-запись из браузера (нужны R2-write creds в env приложения). — Phase 3
- **FR-021** Метаданные портрета: `caption`, `loop_number`, `inspiration` (колонки в mig 121) редактируемы. — Phase 3

## Success Criteria

- **SC-001** ≥120 из 126 артов размещены dry-run’ом без ручной правки. ✅ (123 по анализу)
- **SC-002** «Кватач-Ичл» показывает 4 портрета каруселью с подписями. ✅ код
- **SC-003** Непись с артом и лором открывается в /tg с телефона: арт + читаемая статья. — Phase 2
- **SC-004** DM добавляет новый арт к ноде из приложения за <1 мин. — Phase 3
