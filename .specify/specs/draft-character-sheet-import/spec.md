## Честный порядок (важно прочитать первым)

Ты сформулировал: «RPG движок, начиная с импорта листа персонажа из таблицы
Стасяна». Импорт — правильная **мотивирующая цель** (ежесессионный артефакт —
чарлист — живёт в таблице Стасяна; 0 из 31 листа правились в приложении за месяц,
конституция §«Зачем эпик»). Но технически импорт — **последний** шаг он-рампа, не
первый:

1. **spec-046** (Telegram Auth + Card v0) — вход/shell. *Уже в проде* (см. NEXT).
2. **spec-045 P1** (Engine Core) — модель **модуля** (нода + attachment, R1/R2),
   слой-0 = формулы Стасяна (E9), карточка модуля, эффекты/ресурсы. **Это и есть
   реальный «старт движка».** Драфт есть, ждёт Clarify (C-01…C-05, D-1…D-13).
3. **spec-022** (Player Mobile Mode) — лист как поверхность игры; **US3 = импорт
   xlsx**. Импорт наполняет модель из (2) данными из таблицы Стасяна.

Значит этот черновик = **US3 spec-022**, а не отдельная спека. Ниже — прицельная
проработка импорта, чтобы когда 045-P1 будет готов, импорт был уже продуман.

## Источник импорта (факты, не догадки)

Из `references/README.md` + `memory/character-sheet-excel-system.md`:

- **Источник — `stasyan-sheets-2026-06-10.xlsx`** («Лучшие мальчики для НРИ 2»),
  НЕ lss-json (тот — «not an import source», только референс полей).
- **Каждый персонаж — отдельный лист** (таб). Табы образца: Маркус (монах 9,
  канон-раскладка), Никандр (fighter 8, **дрифт раскладки**), Бальтазар,
  Дамбиниус, Лист5 (пустой).
- **Раскладка Маркуса** (канон): шапка A1–H1 (имя/класс/уровень), статы A3:D8
  (значение+мод+спас), боевые E3:G7 (КД/иниц/скор/HP/кость), навыки A10:D27 (18
  навыков, множитель мастерства 0/0.5/1/2), снаряжение H2:H9+, умения K2:M27
  (название + источник раса/класс/фит), заклинания N13:S19+ (круг/имя/источник),
  ресурсы N2:Q9 (ци/ячейки/вдохновение всего+осталось), валюта E24:G29 **с
  разбивкой `С собой / Дома`**, attunement-колонки у предметов `Предмет/Нуж
  настр/Настр`.
- **Дрифт (Никандр):** полнословные ярлыки («Восприятие»), класс в E1 (не F1),
  нет имени в B1, кость хитов текстом «6д10», послойные слоты `Ячейка/Имеется/
  Осталось`. → **импортёр обязан быть терпимым** (маппинг-с-ревью, не жёсткий парс).
- **Слой-0 формулы** (что НЕ импортируем, а derive-им и сверяем): БМ от уровня,
  мод от стата, навык = мод + FLOOR(БМ×множитель), спас, DC, HP, КД (по классу),
  инициатива, попадание. Полностью в `character-sheet-excel-system.md`.

## Целевая модель (spec-045, не изобретаю)

Из конституции эпика: персонаж = **пирамида модулей** (R1: модуль = нода; R2:
type/instance split; E2: раса/класс-фича/фит/спелл/предмет/бафф — один механизм
«модуль»; E3: MVP модуля = имя+текст+источник, структура опциональна). Слой-0 (E9)
= формулы Стасяна поверх статов/уровня. Импорт наполняет ровно это:

- **PC-нода** ← шапка (имя, класс, раса, уровень).
- **Слой-0 входы** ← статы (6), уровень, множители мастерства навыков, текущий HP,
  остатки ресурсов. Производные (мод/БМ/навыки/спасы/КД/HP/иниц/DC) **derive-ятся
  движком, не импортируются** — и **сверяются** с числами листа (паритет E9/VII).
- **Модули (текст-first, E3)** ← умения K2:M27 (название + источник), заклинания
  (круг+имя+источник), — как модуль-ноды/attachments; структура (эффекты) later.
- **Инвентарь** ← снаряжение H2+ (+ attunement-флаги) → каталог-Образцы (spec-015)
  или free-text; валюта `С собой/Дома` → кошелёк PC (+ «Дома» = ?, см. вопросы).
- **Ресурсы** ← ци/ячейки/вдохновение (всего/осталось) → ресурс-модули (R2 uses на attachment).

## User Scenario (US3 spec-022)

DM загружает xlsx Стасяна → превью-маппинг по табам (каждый таб = PC): что
распозналось, что требует ручного маппинга (дрифт). DM подтверждает → создаются/
обновляются PC-ноды + модули + инвентарь + валюта. Экран паритета: derive-нутые
числа vs числа листа — расхождения подсвечены (E9 «паритет — пол, не потолок»).
Импорт **one-way** (E7): после переезда источник правды — приложение, таблицы
Стасяна параллельным каналом больше нет.

## Открытые вопросы

1. **Порядок подтверждаешь?** 045-P1 → 022 US3 импорт. Или хочешь тонкий
   «импорт-в-текст» ещё до полноценных модулей (импорт как плоский текст-лист,
   структура позже)? *Предложение:* дождаться 045-P1 — иначе импортируем в никуда.
2. **Дрифт-табы** — насколько авто? (а) строгий парс только канон-раскладки
   (Маркус-like), дрифт — ручной маппинг; (б) эвристики + ревью-экран. *Предложение:* (б).
3. **Валюта `С собой / Дома`** — «С собой» → кошелёк PC; «Дома» → ? (общак? второй
   кошелёк? игнор?). Нужно решение.
4. **Снаряжение → каталог или текст?** Матчить H-колонку к Образцам (spec-015) или
   импортировать как free-text? *Предложение:* текст сначала, матч-с-ревью потом.
5. **Идемпотентность** — реимпорт того же листа обновляет или дублирует? (Нужно
   обновление по PC-ключу, как autogen-реконсиляция.)
6. **Паритет-порог** — что делать при расхождении derive vs лист (ручное ✎ на
   листе, формула, которую движок ещё не знает)? Показать, не блокировать (E3/E9).
7. **Заклинания/фиты** — импорт как текст-модули (E3) сразу, привязка к базам
   (spec-047/049) — потом? *Предложение:* да, текст-first.

## Out of Scope

- Сама модель движка (spec-045) — здесь только *импорт в неё*.
- Импорт из longstoryshort.app (lss-json) — не источник (только референс полей).
- Двусторонний синк с таблицей — импорт one-way (E7).
- Базы контента (спеллы/фиты — spec-047/049) — импорт кладёт текст-модули.

## Success Criteria

- **SC-001** DM импортирует xlsx Стасяна; каждый непустой таб → PC-нода с
  модулями, инвентарём, валютой, ресурсами.
- **SC-002** Паритет: derive-нутые движком числа совпадают с листом на канон-табе
  (Маркус); расхождения на дрифт-табе подсвечены, не роняют импорт.
- **SC-003** Реимпорт того же листа обновляет PC, не дублирует.
- **SC-004** После импорта лист живёт в приложении (реалтайм E7); таблица Стасяна больше не нужна как канал.
