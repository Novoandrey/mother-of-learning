## 1. Дайджест контекста

- **Что строим**: тонкий денежный слой поверх готовой бухгалтерии, **внутри `/tg` Mini App** (046), не новый PWA. Кошелёк PC, лента операций, запись расхода/дохода за секунды, переводы PC↔PC и PC↔общак, **свободный общак** (положить/забрать без аппрува — C-05), просмотр общака, realtime (E7).
- **Ставка (SC-007)**: за 30 дней игроцкие транзакции/мес растут с 2 до ≥ 20, либо честный разбор почему нет. Значит запись должна быть **быстрее бумаги** (E6) — это двигатель.
- **Бэкенд не трогаем**, кроме двух осознанных дельт (FR-004): auth-adapter под minted-JWT и флаг свободного общака. Вся бух-логика (резолв монет, гейт статуса, батчи) — на месте.
- **Существующая идентичность** (из 022 §1, `mat-ucheniya/STYLE.md`, `app/globals.css`): Manrope + JetBrains Mono, плоские тени, **табличные цифры**, один акцент. Деньги — это про цифры, так что табличные цифры тут критичны.
- **Платформа-специфика 046**: тема берётся из Telegram `themeParams` (bg/text), дефолт — тёмный neutral; портрет в натуральном соотношении; круглые аватары в списке; никакой установки/манифеста.

## 2. Принципы

Наследуем 022 (P1 стол важнее дивана, P3 бумага прощает — свободный текст первоклассен, P5 тихий UI, P6 одна линза — те же данные/права/действия). Плюс денежные:

- **L1 — Запись за секунды.** От тапа по «+» до записанной операции — обязательные поля только направление + сумма (FR-003). Категория, заметка, день петли — опциональны, с дефолтами. Сравнение — не «лучше десктопа», а «быстрее, чем записать на бумажке».
- **L2 — Честность событий.** Баланс — это реплей append-only транзакций. Оптимистичный апдейт допустим, но при обрыве — честный откат с тостом, без фантомного баланса (Edge Cases). `pending` всегда видимо помечен — игрок понимает, что заявка ждёт ДМа.
- **L3 — Свободный общак, видимый аудит.** Положить/забрать из общака — без трения (C-05). Но каждое движение в ленте общака и в ленте PC, кто инициировал — видно. Контроль через прозрачность, не через запреты (будущий location-gate — отдельно).
- **L4 — Смотреть свободно, менять только своё.** Любого PC можно открыть и прочитать (кошелёк/ленту — E4/FR-005); кнопки записи/перевода появляются только на своих PC.

## 3. Информационная архитектура

### 3.1 Иерархия (внутри `/tg`)

```
/tg
 └─ Список персонажей            ← 046-список, расширен: «Мои» сверху, «Остальные» снизу
     └─ Экран PC (домашняя)      ← портрет + имя + НИЖНИЙ ЛАУНЧЕР АППОВ
         └─ 🛍 Ledger (апп)      ← кошелёк → лента → «+»; переводы и общак отсюда
             ├─ Запись (лоток)
             ├─ Перевод (лоток): на PC / в общак
             ├─ Общак (экран)
             ├─ Балансы всех PC (экран, P2)
             └─ ⋮ Стартовый эквип (P3, «подальше»)
```

### 3.2 D-01 — Лаунчер аппов в PC `[рек.: нижний ряд иконок на домашней PC]`

| Вариант | За | Против |
|---|---|---|
| **A. Нижний ряд иконок-аппов на экране PC** (Ledger=🛍; дальше Лист, и т.д.) ✅ | ровно решение C-04; это и есть нижняя кромка, которую 022 §4.3 *зарезервировал под «и дальше»*; растёт по мере появления аппов; один апп = один полный экран, ментально чисто | пока апп один — ряд из одной иконки выглядит пусто (смягчение: ★ подпись/крупная иконка-кнопка, см. §7) |
| B. Глобальный таб-бар через весь Mini App | привычно | дробит «всё в персонаже» по глобальным табам; ломает «смена PC = зайти в другого PC» (C-02); 022 явно отверг таб-бар как симптом дробления |

Обоснование: лаунчер — per-PC, не глобальный. Когда придёт лист (022) — становится вторым аппом в том же ряду, без переустройства.

### 3.3 D-02 — Навигация внутри Ledger-аппа `[рек.: один скролл + лотки]`

Внутри аппа — **один скроллируемый экран** (кошелёк сверху, лента ниже), «+» запись и переводы — **bottom-sheet лотки**, общак — соседний экран в один тап. Никакого внутреннего таб-бара (P1/L1: scroll over clicks, ≤ 2 тапа до любой операции — FR-002). Совпадает с философией 022 «один документ».

### 3.4 D-03 — Где живёт стартовый эквип `[рек.: за ⋮ в Ledger, разовый]`

«Засунуть подальше» (C-03) = за меню `⋮` Ledger-аппа, пункт «Стартовый эквип». Не на главном пути. После одобрения ДМом — пункт схлопывается в «уже задан» (показывает дату/статус). Разовый вход, не повседневный.

### 3.5 Входные потоки (наследуют 046 + 022 §4.3)

- **0 своих PC**: пусто — «Пока у тебя нет персонажа в этой кампании. Напиши ДМу — он привяжет твоего PC.» (копия 046/022). Но список «Остальные» всё равно показан — смотреть можно.
- **1 свой PC**: 046 уже авто-открывает карточку; здесь — авто-открытие экрана PC (домашней с лаунчером).
- **N своих PC** (Андрей: 3): список, «Мои» группой сверху (последний открытый — первым), «Остальные» ниже. Тап → экран PC.

## 4. Анатомия экранов

### 4.1 Экран PC — домашняя (★ новый)

```
┌──────────────────────────────────────┐
│ ← мои персонажи                       │  ← если N PC; иначе скрыто
│                                        │
│        ┌──────────────────┐            │
│        │                  │            │
│        │    ПОРТРЕТ        │            │  ← натуральное соотношение (046)
│        │   (3:4 плейсх.)   │            │
│        │                  │            │
│        └──────────────────┘            │
│                                        │
│        Миряна Кастиль                  │  ← имя (крупно)
│        Волшебница · петля 4            │  ← ★ сабтайтл (класс/петля — позже)
│                                        │
├──────────────────────────────────────┤
│   ┌────────┐  ┌────────┐  ┌────────┐  │
│   │   🛍   │  │   📋   │  │   ⋯   │  │  ← ЛАУНЧЕР: Ledger / (Лист, серый) / далее
│   │ Деньги │  │  Лист  │  │       │  │
│   └────────┘  └────────┘  └────────┘  │

```
Чужой PC: то же, но лаунчер открывает аппы в **read-only** (L4).

### 4.2 Ledger — главный (★ новый)

```
┌──────────────────────────────────────┐
│ ←            Деньги · Миряна        ⋮ │  ← ⋮ = Балансы всех / Стартовый эквип
├──────────────────────────────────────┤
│  ┌──────────────────────────────────┐ │
│  │  147 зм                          │ │  ← АГРЕГАТ (самое крупное, табл. цифры)
│  │  pp 1   gm 4   sm 7   mm 0        │ │  ← по номиналам (пп/зм/см/мм)
│  └──────────────────────────────────┘ │
│  [ → Перевод ]            [ Общак » ] │  ← ≤ 2 тапа (FR-002)
├──────────────────────────────────────┤
│  Лента                                │
│  ─ п4·д3 ─────────────────────────    │
│   −5 зм  · еда «вино для Ильзы»   ✎   │  ← своё pending → ✎ правка/удаление
│   +30 зм · добыча                     │
│  ─ п4·д1 ─────────────────────────    │
│   ⏳ −12 зм · снаряга      [на одобр.]│  ← pending-бейдж (амбер)
│   …                                   │
│   [ Показать ещё ]                    │  ← пагинация (PostgREST ~1000)
├──────────────────────────────────────┤
│                              ( + )    │  ← FAB записи
└──────────────────────────────────────┘
```

### 4.3 Запись — лоток (дериват десктопной формы)

```
┌──────────────────────────────────────┐
│ Записать операция              ✕      │
│  ( Расход )  ( Доход )                │  ← направление (обяз.)
│  Сумма:  [    5    ] зм               │  ← обяз.
│  ─────────────────────────────        │
│  Категория: [ еда ▾ ]      (опц.)     │
│  Заметка:   [ вино для Ильзы ](опц.)  │
│  День: [ петля 4 · день 3 ] ✎         │  ← дефолт по правилам, override
│         ↳ предмет?  [+ предмет]       │  ← переключение money↔item
│            [ Записать ]               │
└──────────────────────────────────────┘
```
Item-режим: поле «Название» + (опц.) связь с образцом из базы + кол-во. Хоумбрю = свободный текст (P3/L-бумага).

### 4.4 Перевод — лоток

```
┌──────────────────────────────────────┐
│ Перевод                        ✕      │
│  Кому:  ( PC ▾ )   ( в Общак )        │
│  Сумма: [   10   ] зм                 │
│  Заметка: [            ] (опц.)       │
│            [ Перевести ]              │
└──────────────────────────────────────┘
```
В общак → свободно (C-05), без «на одобрение». На PC → если ты не ДМ, уходит в pending (C-01).

### 4.5 Общак — экран

```
┌──────────────────────────────────────┐
│ ←                    Общак            │
│   ┌────────────────────────────────┐  │
│   │  860 зм                        │  │  ← баланс общака
│   │  pp 0  gm 86  sm 0  mm 0       │  │
│   └────────────────────────────────┘  │
│   [ Положить ]        [ Забрать ]     │  ← оба свободны (C-05)
│   Движения                            │
│    +10 зм · Миряна · п4·д3            │
│    −50 зм · Зак · п4·д2              │
│    …                                  │
└──────────────────────────────────────┘
```

### 4.6 Балансы всех PC — экран (P2, прозрачность E4)

Простой список: имя PC · агрегат зм, по убыванию; общак отдельной строкой сверху. Read-only. Тап по строке → экран того PC (read-only, L4).

### 4.7 Стартовый эквип — экран (P3, за ⋮)

```
┌──────────────────────────────────────┐
│ ←            Стартовый эквип          │
│  Собери стартовый набор. ДМ одобрит.  │
│  Предметы                             │
│   • Кинжал ×1                    ✕    │  ← из базы (typeahead) или
│   • Костюм чирлидерши ×1         ✕    │     свободный текст (хоумбрю)
│   [ + добавить предмет ]              │
│  Деньги                               │
│   • +15 зм «стартовый кошелёк»   ✕    │
│   [ + добавить деньги ]               │
│  ───────────────────────────────     │
│            [ Отправить ДМу ]          │  ← один submitBatch → очередь (C-03)
└──────────────────────────────────────┘
```
После отправки: «Отправлено, ждёт одобрения · п4·д1». После одобрения: пункт в ⋮ становится «Задан ✓».

## 5. Состояния

- **Загрузка**: спиннер/«Загрузка…» (как 046).
- **Пусто — нет своих PC**: копия выше; список «Остальные» всё равно есть.
- **Пусто — нет операций**: «Пока пусто. Запиши первую операцию по +.» (★).
- **Ошибка сети при записи (L2)**: оптимистичную строку откатываем, тост «Не сохранилось — нет сети. Попробуй ещё раз.» Баланс не врёт.
- **Бэкенд отклонил правило** (овердрафт без размена и т.п.): показываем честный текст ошибки из экшена, ничего молча не теряем.
- **pending**: строка с ⏳ + бейдж «на одобрение» (амбер); своё pending редактируемо/удаляемо (C-01).
- **Realtime-приход (E7)**: новая строка втекает сверху, баланс пересчитывается; ★ один тихий момент «праздника» — лёгкая подсветка строки на 1 сек (единственная анимация, P5).
- **Read-only (чужой PC)**: «+»/переводы/правка скрыты; видно кошелёк+ленту.

## 6. D-04 — Тема и токены `[рек.: Telegram themeParams + тёмный дефолт, токены 022]`

- **Источник темы**: Telegram `themeParams` (bg/text/hint/link) → переменные страницы; дефолт — тёмный neutral (как 046 сейчас). Не навязываем светлую «тёпло-белую» из 022 — Mini App уважает клиента.
- **Типографика**: Manrope (текст) + JetBrains Mono **табличными цифрами** для всех сумм и номиналов (022 §1). Деньги без табличных цифр прыгают — недопустимо.
- **Акцент**: проектный синий (022 §10) для действий; **амбер** — единственный статус-цвет, только `pending`.
- **Плотность/тени**: плоско, как 022; карточка кошелька — единственный «приподнятый» элемент.
- **Токены не дублирую**: источник — `022 design.md §10` + `mat-ucheniya/STYLE.md` + `app/globals.css`. Дельта 044 — только: bg/text из Telegram, амбер=pending.

## 7. ★ Креативные слоты

- **Иконка Ledger-аппа** — мешочек 🛍 (подтверждено Андреем «иконка мешочка»). Финальный глиф/стиль — твой.
- **Подпись аппа** — «Деньги»? «Кошелёк»? «Casus»? (флейвор кампании). v0 = «Деньги».
- **Пустые состояния** — тексты выше — болванки, перепиши под голос кампании.
- **Момент realtime-прихода** — единственная анимация продукта (P5): какой именно «праздник» при втекании чужой транзакции/одобрения.
- **Микрокопия** — вся русская копия v0, твоя.
