# Claude Design — промпт: Mobile Ledger (3 экрана)

> Самодостаточный промпт для Claude Design (🌐). Дистиллят `../design.md`.
> Андрей прогоняет его, экспорт (JSX-фреймы / CSS / скрины) → `./export/`.
> Рисуем **только 3 реально новых экрана**; остальные поверхности 044 —
> мобильные рендеры готовых десктоп-флоу, отрисовка не нужна.

---

## Контекст продукта

Telegram Mini App для настольной D&D-кампании (~20 игроков). Это **денежный апп персонажа**: игрок открывает своего PC и ведёт кошелёк — баланс, лента операций, быстрая запись траты/дохода, общий фонд («общак»). Цель — записать операцию **быстрее, чем на бумажке**; за столом, в полумраке, одной рукой, ДМ ждёт. Тихий интерфейс, читаемость с вытянутой руки, важные числа — самые крупные.

Экран открывается внутри Telegram, без установки. Это мобайл-фёрст, портрет, логическая ширина ~390 px.

## Тема и токены

- **Тёмная тема** (нейтральный тёмный фон, светлый текст). На проде цвета берутся из Telegram `themeParams`; для макетов используй тёмный нейтрал: фон ~`#0a0a0a`/`#171717`, текст ~`#f5f5f5`, приглушённый текст ~`#a3a3a3`.
- **Типографика**: Manrope для текста; **JetBrains Mono с табличными цифрами** для всех сумм и номиналов монет (числа не должны прыгать).
- **Акцент**: один синий для действий/кнопок.
- **Статус-цвет**: единственный — **амбер**, только для состояния «на одобрение» (`pending`).
- **Стиль**: плоский, без декоративных теней. Единственный «приподнятый» элемент — карточка баланса кошелька. Углы скруглённые (12–16 px). Минимум контролов — каждая кнопка оправдана.
- **Одна анимация на весь продукт**: лёгкая подсветка строки на ~1 сек, когда в ленту втекает новая операция. Больше нигде анимаций нет.

## Монеты

Номиналы: **пп** (платина) · **зм** (золото) · **см** (серебро) · **мм** (медь). Баланс показывается как крупный агрегат в зм + строка из 4 номиналов.

## Экран 1 — Домашняя персонажа (с лаунчером аппов)

Назначение: «капитанский мостик» одного PC. Сверху — большой портрет в натуральном соотношении (плейсхолдер 3:4, если портрета нет — серый блок с первой буквой имени). Под портретом — имя крупно, ниже приглушённый сабтайтл («Волшебница · петля 4»). Внизу экрана — **ряд иконок-аппов** (лаунчер): первый и единственный активный — **«Деньги»** с иконкой мешочка; рядом неактивная серая «Лист» (придёт позже) и место под будущие. Тап по «Деньги» → Экран 2.

Если у игрока несколько своих персонажей — сверху мелкая ссылка-назад «← мои персонажи».

Состояния:
- активный апп (мешочек, цветной) vs будущий апп (серый, неактивный, с подписью);
- портрет есть / портрет-плейсхолдер;
- свой PC (аппы активны) vs **чужой PC** (открывается, но read-only — но это поведение Экрана 2, на домашней разницы нет, кроме того что чужого можно открыть свободно).

Акцент дизайна: лаунчер не должен выглядеть пустым при одной активной иконке — сделай активную иконку-кнопку крупной и приятной, будущие — деликатно-приглушёнными.

## Экран 2 — Ledger: кошелёк + лента + запись

Назначение: главный экран денежного аппа. Один скроллируемый экран.

Сверху — апп-бар: «← Деньги · Миряна», справа «⋮» (меню: «Балансы всех», «Стартовый эквип»).

Блок кошелька (единственная приподнятая карточка): **крупный агрегат** «147 зм» (самое большое число на экране, табличные цифры), под ним строка номиналов «пп 1 · зм 4 · см 7 · мм 0». Под карточкой — два компактных действия в ряд: «→ Перевод» и «Общак »».

Ниже — **Лента операций**, сгруппированная разделителями по дню петли («п4 · д3»). Строка операции: знак и сумма («−5 зм», «+30 зм»), категория и заметка («еда «вино для Ильзы»»), тонко — кто инициировал, если не ты. Свои операции в статусе `pending` помечены значком ⏳ слева и амбер-бейджем «на одобрение» справа; свои pending-строки имеют иконку правки. Внизу ленты — «Показать ещё» (пагинация). Доходы/расходы различимы спокойно (не кричащим зелёным/красным — приглушённо).

Внизу справа — плавающая кнопка **«+»** (записать операцию).

Состояния ленты: обычная; пустая («Пока пусто. Запиши первую операцию по +.»); строка `pending` (амбер); только что втёкшая строка (1-сек подсветка); read-only-режим (чужой PC — нет «+», нет правки).

### Лоток записи (bottom-sheet к Экрану 2)

Выезжающая снизу панель. Поля сверху вниз: переключатель **«Расход / Доход»** (обязательно), поле **суммы** с единицей «зм» (обязательно), затем опционально — категория (выпадашка), заметка (текст), «День: петля 4 · день 3» с иконкой правки (дефолт проставлен). Мелкая ссылка «+ предмет» переключает в режим предмета (поле названия + количество; название можно ввести свободным текстом). Внизу — кнопка «Записать». Минимум обязательного — направление и сумма; всё прочее по желанию.

## Экран 3 — Стартовый эквип (разовый, спрятан за ⋮)

Назначение: игрок один раз собирает стартовый набор и отправляет ДМу на одобрение. Не повседневный экран.

Заголовок «← Стартовый эквип», подпись «Собери стартовый набор. ДМ одобрит.» Две секции-списка:
- **Предметы**: строки «• Кинжал ×1 ✕», «• Костюм чирлидерши ×1 ✕» (предмет либо выбран из базы через typeahead, либо введён свободным текстом — оба варианта выглядят одинаково в списке); кнопка «+ добавить предмет».
- **Деньги**: строки «• +15 зм «стартовый кошелёк» ✕»; кнопка «+ добавить деньги».

Внизу — кнопка «Отправить ДМу».

Состояния: пустой набор (обе секции пусты, кнопка отправки неактивна); набор заполнен; после отправки — экран сменяется на спокойное «Отправлено, ждёт одобрения · п4 · д1»; (одобренное состояние — отдельно, можно не рисовать).

---

## Что НЕ рисуем (готовые паттерны / 022)

Лоток перевода, экран общака, экран «Балансы всех PC», список персонажей «Мои/Остальные», read-only-вариант чужого PC — это вариации десктоп-флоу и уже спроектированного в spec-022; в этот промпт не входят.

## Выдача

По одному фрейму на каждый из 3 экранов (для Экрана 2 — отдельный фрейм лотка записи). Тёмная тема, ~390 px ширина, реальная русская микрокопия из брифов выше (правится свободно). Экспорт положи в `./export/`.
