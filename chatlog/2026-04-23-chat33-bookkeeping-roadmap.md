# Chat 33 — Бухгалтерия roadmap, 2026-04-23

## Контекст (откуда пришли)

Пользователь принёс сырой дамп из телеграма с обсуждением "лут-трекинга
в циклах времени". Попросил структурировать и спланировать. Изначально
я бросился кодить (миграция 032 + LoopProgressBar компонент) — пользователь
остановил и сказал, что чат чисто для спеки/плана, без кода.

Цель чата: **roadmap без кода**.

## Что сделано

- Проанализирован дамп — выделены ~10 подфич, три из них отсечены как
  нерелевантные/преждевременные (полный inventory, автобой→автолут,
  realtime).
- Согласовано название: **Бухгалтерия**.
- Принято решение разбить фичу на **7 отдельных спецификаций**
  (`spec-009` … `spec-015`), а не одну большую.
- Зафиксированы ключевые решения:
  - валюта — GP (DECIMAL 0.01), номиналы cp/sp/gp/pp как отдельные
    стопки в инвентаре, при списании мельчайшие первыми
  - апрув-флоу — до spec-014 всё auto-approved (правило переноса
    конституции VII); spec-014 меняет контракт (игрок больше не
    пишет напрямую, только батчи → ДМ апрувит)
  - кредиты — НЕ отдельная сущность (в петле не возвращаются),
    живут как `category='credit'` в транзакциях
  - общак сбрасывается с петлёй (wipeable state, принцип I)
- Разобрана **временная модель** с учётом реальности игры:
  - 29 PC, пачки 4-7, некоторые пропускают петлю целиком
  - сессии могут пересекаться во времени свободно
  - три понятия "текущий день": **loop frontier** / **character
    frontier** / **session day**
  - retrograde = soft warning, не блокер
- Spec-009 расширен: включает `session.participants` (пачка), не
  только `day_from`/`day_to`.
- Спеки 007/008 — in-prod без формальных папок, решено **не**
  делать retrospective backfill.
- Установлен протокол: **каждая спека в отдельном чате**.
