### Phase 1 — миграция 043 (chat 55)
- `node_types` upsert для `'item'` per existing campaign.
- `item_attributes` table: PK на `node_id`, FK `ON DELETE CASCADE`,
  hot columns (category_slug, rarity, price_gp, weight_lb,
  slot_slug, source_slug, availability_slug), 5 индексов
  (composite + 4 partial), CHECK на rarity enum + non-negative
  price/weight, RLS read-by-member.
- `transactions.item_node_id` nullable FK `ON DELETE SET NULL`,
  partial index `where item_node_id IS NOT NULL`,
  CHECK `item_node_id IS NULL OR kind = 'item'`.
- Categories scope CHECK расширен с 2 до 5 значений.
- Per-campaign seeds через `cross join` + `on conflict do nothing`:
  8 категорий, 13 слотов, 2 источника, 4 availability.
- Идемпотентно (ADD COLUMN IF NOT EXISTS, DROP CONSTRAINT IF
  EXISTS, ON CONFLICT DO NOTHING).
- T004: `'item'` добавлен в `NODE_TYPES` в `lib/seeds/dnd5e-srd.ts`,
  новый `lib/seeds/item-value-lists.ts` для seed'ов в новых
  кампаниях, hooked в `initializeCampaignFromTemplate`.

### Phase 6 — typeahead retrofit (chat 59–63, самая большая)

**Server-side** (chat 59–60):
- `CreateTransactionInput.itemNodeId?` — резолвит каноничное
  имя через `getItemById` (FR-014 snapshot).
- `updateTransaction.itemNodeId` — поддерживает add/change/remove
  (set null/'' для unlink); очищает при kind=`'money'`.
- `ItemTransferInput.itemNodeId?` — apply на ОБЕ legs;
  ownership check теперь через `item_node_id` когда есть, иначе
  strict `item_name AND item_node_id IS NULL`.
- `BatchRowSubmitInput`: `'item'` и `'transfer-item'` оба
  принимают `itemNodeId`.
- `ItemStashInput.itemNodeId?` в `app/actions/stash.ts`,
  пробрасывается в `createItemTransfer`.

**`<ItemTypeahead>` компонент** (chat 60, ~210 LOC):

- «образец» badge когда linked.

- Player free-text hint когда no match.
- Click-outside collapse, Escape clear.
