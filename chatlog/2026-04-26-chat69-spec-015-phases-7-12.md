### Phase 7 — InventoryTab (chat 67)

- Новый `<InventoryTab>` (server, async) + `<InventoryTabControls>`
  (client island). URL-driven `?tab=inventory&loop=N&day=M&group=...`
  через `router.replace` (history-friendly). Loop picker фильтрует
  past + current (FR-023b). Day input commit'ится onBlur. Slice-чип
  «Срез: петля N · день M». Hot-field chips (category/rarity/slot/
  price/weight) только для linked rows; rarity-цвет в тон каталогу.
- `lib/items-grouping.ts:groupInventoryRows` — pure helper параллельно
  `groupItems`. Free-text rows (`attributes === null`) собираются в
  хвост buckета `__no_link__` «Без образца». 6 vitest-кейсов.
- PC page (`/c/[slug]/catalog/[id]`) принимает `searchParams`,
  рендерит 2-tab nav (Кошелёк / Инвентарь) для character-нод.
  Lazy-load `getLoops` только при `?tab=inventory`.
- Stash page (`/c/[slug]/accounting/stash`) заменил `<InventoryGrid>`
  на `<InventoryTab>` внутри `StashPageTabs.itemsContent`. Stash
  default day = 30 (нет session frontier).

### Phase 9 — SRD seed (chat 67)

- **Decision:** Option C — 50 hand-curated entries (vs Option B
  parse open5e ~400). ROI на backfill доминируется common items;
  no licensing/parse-fragility blocker этой сессии.
- `lib/seeds/items-srd.ts` — 50 entries: 15 weapons, 8 armour,
  12 gear/tools, 8 consumables, 7 magic. Categories/slots/rarities
  сходятся с дефолтными seed'ами мига 043.
- `scripts/items-srd-codegen.ts` — TS regenerator, читает
  `ITEMS_SRD_SEED`, эмитит SQL в stdout. Команда:
  `npx tsx scripts/items-srd-codegen.ts > supabase/migrations/044_srd_items_seed.sql`
- `supabase/migrations/044_srd_items_seed.sql`:
  - **Phase 1 per-campaign:** `WITH seed(...) AS (VALUES <50>)` →
    `INSERT INTO nodes WHERE NOT EXISTS (campaign_id, srd_slug)` →
    `RETURNING id+srd_slug` → `INSERT INTO item_attributes` joined на
    srd_slug. `source_slug='srd-5e'`.
  - **Phase 2 per-campaign:** `UPDATE transactions SET item_node_id`
    matching seeded node WHERE link is NULL AND
    `LOWER(TRIM(item_name)) = LOWER(TRIM(title)) OR LOWER(srd_slug)`.
    Restricts to `ia.source_slug='srd-5e'`.
  - `RAISE NOTICE` per campaign с insert + backfill counts (FR-029).
  - Idempotent: re-run не дублирует, не re-point'ит уже-link'нутые.

### Phase 12 — Smoke + close-out (chat 69)

- `scripts/check-rls-015.sql` — 6 кейсов, обёрнутых в
  `BEGIN ... ROLLBACK`:
  1. RLS-1: outsider видит 0 строк `item_attributes`.
  2. RLS-2: owner кампании A видит 1.
  3. CASCADE-1: DELETE node → `item_attributes` row gone.
  4. CASCADE-2: DELETE node → `transactions.item_node_id = NULL`,
     транзакция жива (`ON DELETE SET NULL`).
  5. CHECK-1: `(kind='money', item_node_id NOT NULL)` отвергается.
  6. CHECK-2: 5 валидных scope'ов принимаются, чушь отвергается.
  Final summary: PASS counter vs total=6, RAISE EXCEPTION на mismatch.
