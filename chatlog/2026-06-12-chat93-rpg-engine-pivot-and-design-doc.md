## Что помнить следующему чату
**Суть пивота**: персонаж = пирамида модулей по уровням (раса / класс-уровни /
фиты / предметы / баффы). Модуль декларирует эффекты на атрибуты — одна
система вместо хардкода спеллов (game programming patterns: Component,
Type Object, Prototype=форк, data-driven). MVP модуля = текстовое поле без
эффектов (валидный модуль с effects[]=∅) — лист живёт сразу, структура
прорастает постепенно. Сложные свойства могут навсегда остаться текстом.
Формулы Стасяна не выбрасываются — становятся слоем-0 деривации
(мод/БМ/навык), эффекты складываются поверх. Пилоты: Каэл, Миряна, Британия.
Канон-ноды «прибиты» (read-only), хоумбрю — через форк и конструктор.
**Mobile по-прежнему first.** design.md переживает ~80% (кубы, импорт,
анатомия листа, токены); сносится модель данных FR-004/005 и направление Plan.

**Черновик разбиения эпика** (выделить/оценить/упорядочить — задача след. чата):
1. Аудит трёх пилотов → опись всех свойств/предметов/спеллов/кулдаунов =
