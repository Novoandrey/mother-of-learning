# Chat 78 — концепт-редактор поверх sandbox (план), 2026-04-28

## Контекст

Параллельно текущей очереди приоритетов (инвентарь → DB security →
item search → наполнение истории транзакций 5-й петли) пользователь
хочет зафиксировать план под отдельную сущность: «концепты для
локаций, монстров, квестов» — markdown-редактор, картинки, ссылки
между объектами, разделение игровых статов от прозы, всё это в
draft-режиме до утверждения главным DM.

Часть требований уже закрыта инфраструктурно (zettelkasten = nodes +
edges + JSONB fields, тип создаётся из UI без миграций; базовый
markdown-редактор лежит в `components/markdown-content.tsx`).
Незакрытое: visibility-флаг (это spec-020 «DM sandbox» в backlog'е),
storage для картинок, wikilinks `[[Title]]`, структурированные поля
per node_type.

## Что сделано

- `backlog.md` spec-020: расширен 4-мя слоями концепт-редактора
  поверх существующего sandbox-плана (storage, wikilinks,
  структурированные поля, editor v2 опц.). Добавлены первые
  тестовые концепты (Декан Сиори ← Community; бар Золотой Петух ←
  MacLaren's из HIMYM) и блок «Зависимости» (ждёт после
  инвентаря/security/search/5й петли). Отдельный пункт «Без
  миграций по команде» отмечает что архитектурно уже работает,
  открытый вопрос — как Claude/другие DM дотягиваются до БД (CLI
  по service role / MCP / server actions).
- `Updated:` строка в backlog.md обновлена.
- `NEXT.md` не трогаем — «в проде» и «следующий приоритет» не
  изменились (текущий фокус всё тот же).

## Решения

- Концепт-редактор не отдельная спека — это расширение скоупа
  spec-020 (sandbox + редактирование внутри sandbox = одна фича).
- Порядок слоёв внутри спеки: visibility (база) → storage →
  wikilinks → структурированные поля. Editor v2 — опционально,
  низкий приоритет.
- Тестовые концепты пока не создаются (нет sandbox = утечка
  игрокам), сидят в backlog как маркер «с чего начнём наполнение».
