# Changelog

User-facing log изменений «Мать Учения». Версии нумеруются по
количеству чатов разработки (a.b.c, где c = chat count).

## 0.4.25 — апрель 2026

Закрыта серия 009→015: бухгалтерия, учёт лута, общак, заявки,
каталог предметов. Что появилось в кампании:

### 📅 Прогрессбар петли + сессии-как-пачки (spec-009)

- Сессия теперь имеет диапазон дней (`day_from`–`day_to`) и список
  участников.
- На вкладке «Петли» — визуальный прогрессбар: видно где в петле
  какие пачки играли, где наложения.

### 💰 Бухгалтерия: лента транзакций (spec-010)

- Новая вкладка «Бухгалтерия». Любой member пишет: доход / расход
  / перевод / предмет.
- Баланс PC = сумма транзакций. Виден на карточке персонажа
  (Wallet block) и в общей ленте.
- Категории редактируются ДМом (Income / Expense / Loot / Credit
  / Other / …).
- Фильтры: по PC, петле, дню, категории, kind. Сворачиваемая
  панель.

### 🏛️ Общак (spec-011)

- Одна общая «нода-кошелёк» на кампанию: деньги + предметы.
- Кнопки `Положить в общак` / `Взять из общака` рядом с балансом
  PC и в actor bar — один тап.
- Если у PC не хватает на расход — форма предлагает добрать из
  общака.
- На странице общака — две вкладки (предметы + лента), общий
  ledger со скрытым actor.

### ⚙️ Стартовый сетап петли (spec-012)

- ДМ задаёт «как стартует петля»: сколько денег у каждого PC,
  кредиты, что в общаке, стартовые предметы.
- Один клик «Применить стартовый сетап» — все нужные транзакции
  пишутся пачкой.
- Ручные правки сохраняются: реран сетапа не затирает то, что ДМ
  поправил руками.

### ⚔️ Лут энкаунтеров (spec-013)

- На странице энкаунтера — панель «Распределение лута»: монеты +
  предметы + получатели (PC / общак / split поровну).
- Один клик «Применить» — пишет все транзакции, привязывает к
  энкаунтеру через mirror-ноду.
- Реранится без дублей; ручные правки лута игнорируются (как в
  spec-012).

### 📋 Заявки на транзакции (spec-014)

- Игроки могут подавать pending-транзакции — ДМ видит очередь и
  одобряет / отклоняет (одной строкой или пачкой).
- Бейдж с количеством ожидающих заявок на вкладке «Бухгалтерия» у
  ДМа.
- Игрок может отозвать свою заявку до approval.
- Approved строки попадают в баланс; rejected/pending — нет.

### 🎒 Каталог предметов (spec-015)

- Новая вкладка 🎒 «Предметы»: фильтры (категория / редкость /
  слот / цена / источник / доступность), группировки, поиск, клик
  по строке раскрывает описание.
- 91 SRD-предмет уже в базе: оружие, броня, расходники,
  магические предметы (+1/+2 экипировка, жезлы, wondrous slot
  items, зелья).
- Образец предмета — отдельная карточка с атрибутами и историей
  всех связанных транзакций.
- Тайпахед в форме транзакции / лута энкаунтера: выбираешь
  Образец — он линкуется к транзакции, история собирается
  автоматически.
- Инвентарь PC и общака с фильтрами по дню/петле, группировки.
- Settings: ДМ редактирует свои таксономии (категории / слоты /
  источники / доступность) + задаёт цены по умолчанию по
  редкости (отдельно для магических и расходников). Авто-префилл
  цены при создании Образца.
