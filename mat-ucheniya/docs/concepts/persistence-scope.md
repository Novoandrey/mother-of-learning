# [draft] Persistence scope — что переживает петлю

> Заглушка. Содержание будет наполняться постепенно.

Каждое событие имеет тег `persistence_scope`: `loop` (состояние мира, инвентарь, сюжетные линии — сбрасывается при reset петли), `character` (память персонажа, уровни, навыки — сохраняется), `meta` (мета-сюжет, NPC-память «кто-то помнит» — сохраняется). Loop reset = создание нового `loop_id`; при свёртке мира фильтр оставляет только текущий loop + character/meta-events.

## Что планируется в статье

- Линейная модель: одна активная петля одновременно
- Прошлые петли read-only, доступны для просмотра лога
- Branching на будущее: поле `branch_id` default 'main'
- Что в проде сейчас (loop как нода, ручная привязка через day_in_loop)
