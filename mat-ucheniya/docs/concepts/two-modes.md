# [draft] Игрок vs DM — два режима

> Заглушка. Содержание будет наполняться постепенно.

Приложение обслуживает двух принципиально разных пользователей поверх одних и тех же данных. Игроку нужна книга-помощник: читалка, mobile-first, минимум кнопок, потребление информации. DM нужна панель управления: desktop-primary, плотный UI, конфигурация мира.

## Что планируется в статье

- Перечисление потребностей игрока (лист персонажа, история, инвентарь, петля)
- Перечисление потребностей DM (мир, NPC, энкаунтеры, лут, бухгалтерия)
- Один датасет — два UX, как это держится в коде
- Mobile-first для игрока vs desktop-primary для DM (контракт)
