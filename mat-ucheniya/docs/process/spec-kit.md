# [draft] Spec-kit workflow

> Заглушка. Содержание будет наполняться постепенно.

Адаптированная для solo-разработки версия GitHub spec-kit: пять фаз — Specify (что и зачем) → Clarify (вопросы и ответы) → Plan (как реализовать) → Tasks (последовательность шагов) → Implement (по одной задаче). Артефакты `spec.md`, `plan.md`, `tasks.md` коммитятся в `.specify/specs/NNN-name/`. Жёсткое правило: не прыгать через фазы; каждая ждёт явного «ok» от пользователя.

## Что планируется в статье

- Каждая фаза: вход, выход, кто пишет
- Шаблоны артефактов (где лежат, как использовать)
- Per-session chatlog: что класть в файл, как именовать
- Что считается одной спекой (правило размера)
- Когда отступать от spec-kit (мелкие баги, точечные фиксы)
