# Версии и CHANGELOG

> Заглушка. Содержание будет наполняться постепенно.

Полу-семвер: a.b.c, где c — счётчик чатов разработки, b — минор-версии (новая фича/раздел), a — мажор (большие пивоты). `CHANGELOG.md` ведёт user-facing хронологию по версиям. `NEXT.md` — текущее состояние («в проде» + «следующий приоритет»). Bump происходит в конце сессии вместе с обновлением обоих файлов.

## Что планируется в статье

- Когда patch, когда minor, когда major
- Что пишем в CHANGELOG (player-facing changes), что нет
- NEXT.md и его секции: «В проде сейчас» / «Следующий приоритет»
- Версия в package.json — синхронизация
