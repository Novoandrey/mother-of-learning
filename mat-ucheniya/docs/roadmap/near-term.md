## Активная работа

### spec-052 — Inventory: контейнеры, наборы, статус «Надето»

**Статус**: код готов (T001–T029), ветка
`claude/052-inventory-containers-sets`, гейт зелёный; ждёт T030–T032 —
staging + миграции 118/119/120 + E2E, затем PR в `main`.

Следующая спека поверх выкаченного мобильного кошелька (spec-044).
Расширяет инвентарь тремя новыми концептами:

- **Контейнеры** — передача предметов между персонажами и из/в
  общак через UI «забрать/положить».
- **Покупка за голду** — трата золота и одновременный приход предмета
  в одно действие.
- **Наборы** — «купить стартовый набор» создаёт сразу несколько
  предметов и списывает деньги одной кнопкой.
- **Статус «Надето»** — новый флаг на позиции инвентаря; влияет на
  будущий movement-бонус от снаряжения.

Плюс фидбэк прода (chat 98): свои pending-заявки с отменой (FR-015),
«количество» как первоклассное поле везде (FR-016, C-12).

Бухгалтерский backend не трогает: ходы = `item-tx`, покупка =
`money-tx + item-tx`; только добавляется флаг «Надето» и механика
наборов. Спека: `.specify/specs/052-inventory-containers-sets/spec.md`.

## Зависимости: почему именно такой порядок

1. **spec-052** идёт первым — короткий, высокая ценность для игроков
   без зависимостей от движка.
2. **spec-045** Engine Core — фундамент для листа персонажа. Без него
   spec-022 v3 строить не на чём.
3. **spec-022 v3** — первый «лист в продукте» вместо таблицы Стасяна;
   цель: листы начнут жить в приложении (SC-4 из spec-022).
4. **Роадмап 030+** начинается параллельно с эпиком там, где нет
   прямой зависимости (030 портреты, 038 часы/проекты).
5. **spec-031 Карта** — фундамент для движения пачки (035/036) и
   encounter rework (032). До карты encounter rework бессмысленен.

→ [`engine-pivot.md`](engine-pivot.md) — полная мотивация и принципы
эпика движка.
