# [draft] Летопись

> Заглушка. Содержание будет наполняться постепенно.

Markdown-блоки на нодах + хронологический лог записей. У каждой ноды (сессия, локация, NPC) есть редактируемое markdown-поле — DM или владелец туда пишет описание. Записи летописи — отдельный лог `chronicles` со ссылкой на ноду и dayInLoop. Автосохранение черновиков через localStorage (баннер «найден несохранённый черновик»).

## Что планируется в статье

- Markdown-блок: рендеринг, режим редактирования, save/cancel
- Chronicles: создание записи, привязка к ноде
- Автосохранение: ключи localStorage, восстановление, чистка
- Что считается черновиком (pristine state не пишется)
