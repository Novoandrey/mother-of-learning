# Фичи

> Заглушка. Содержание будет наполняться постепенно.

Описание фич приложения по одной папке на каждую. Внутри папки: `README.md` объясняет, что это и как пользоваться (для всех), `technical.md` — что под капотом (для разработчиков, добавляется только если есть что класть). Сложные фичи (бухгалтерия, инвентарь) имеют дополнительные файлы для подфич.

## Что планируется в статье

- Карта зависимостей между фичами (что от чего зависит)
- Список фич с однострочной мотивацией
- Что считается фичей, а что — инфраструктурой
