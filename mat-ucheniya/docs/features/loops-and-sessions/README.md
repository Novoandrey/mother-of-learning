# Петли и сессии

> Заглушка. Содержание будет наполняться постепенно.

Петля — нода типа `loop` с последовательным номером и длиной в днях. Сессия — нода типа `session` с диапазоном дней внутри петли и списком участников. Прогрессбар на вкладке «Петли» визуализирует, какие пачки играли в какие дни, где наложения. Все события приложения привязаны к (loop_number, day_in_loop).

## Что планируется в статье

- Создание петли: длина, начало, перенос характеристик
- Создание сессии: участники, диапазон дней
- Frontier — текущий день петли
- Прогрессбар и наложения
- Ссылка из транзакции на сессию (как привязывается)
